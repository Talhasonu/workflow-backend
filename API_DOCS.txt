================================================================
            WORKFLOW BACKEND - API DOCUMENTATION
================================================================
Base URL : http://localhost:5000/api/v1
Auth     : Bearer token required where marked [AUTH]
Role     : [SUPERADMIN] means only superAdmin role allowed
================================================================


================================================================
  1. USERS
================================================================
Route Prefix: /api/v1/users

----------------------------------------------------------------
1.1  SIGN UP
----------------------------------------------------------------
Method : POST
Route  : http://localhost:5000/api/v1/users/signup
Auth   : No

Body (JSON):
{
  "email": "john@example.com",
  "name": "John Doe",
  "password": "Password@123",
  "confirmPassword": "Password@123"
}

----------------------------------------------------------------
1.2  VERIFY EMAIL
----------------------------------------------------------------
Method : POST
Route  : http://localhost:5000/api/v1/users/verify-email/:token
Auth   : No

Params:
  token  - the verification token sent to email (in URL)

Body   : None

----------------------------------------------------------------
1.3  SIGN IN
----------------------------------------------------------------

Route  : http://localhost:5000/api/v1/users/signin
Auth   : No

Body (JSON):

}

----------------------------------------------------------------
1.7  INVITE TEAM MEMBER
----------------------------------------------------------------
Method : POST
Route  : http://localhost:5000/api/v1/users/invite-team-members
Auth   : [AUTH] [SUPERADMIN]

Headers:
  Authorization: Bearer <token>

Body (JSON):
{
  "name": "Jane Doe",
  "email": "jane@example.com",
  "role": "editor",        // "admin" | "viewer" | "editor"

}

----------------------------------------------------------------

----------------------------------------------------------------
Method : PATCH
Route  : http://localhost:5000/api/v1/users/update-team-members/:userId
Auth   : [AUTH] [SUPERADMIN]

  "role": "viewer",        // "admin" | "viewer" | "editor"

----------------------------------------------------------------
1.9  GET WORKSPACE USERS
----------------------------------------------------------------
Method : GET
Route  : http://localhost:5000/api/v1/users/workspace-users
Auth   : [AUTH]

Headers:
  Authorization: Bearer <token>

Query Params (optional):
  page   - page number, default: 1
  limit  - items per page (1-100), default: 10
  role   - filter by role: "admin" | "editor" | "viewer"
  search - search by name or email

Example:
  http://localhost:5000/api/v1/users/workspace-users?page=1&limit=10&role=editor&search=john

================================================================
  3. PROCESS
================================================================
Route Prefix: /api/v1/process

----------------------------------------------------------------
3.1  CREATE PROCESS
----------------------------------------------------------------
Method : POST
Route  : http://localhost:5000/api/v1/process/create
Auth   : [AUTH] [SUPERADMIN]

Headers:
  Authorization: Bearer <token>

Body (JSON):
{
  "name": "Onboarding Process",
  "visibility": "private",       // "private" | "public"
  "steps": [
    {
      "title": "Send Welcome Email",
      "description": "Send the welcome email to the new hire",   // optional
      "timeEstimate": "30 mins",                                  // optional
      "assignee": "64f1a2b3c4d5e6f7a8b9c0d1",                    // optional, userId
      "notes": "Use email template v2",                           // optional
      "status": "draft"                                           // optional: "draft"|"completed"|"inprogress"
    }
  ],
  "description": "Process for new employee onboarding",          // optional
  "category": "HR",                                              // optional
  "assignees": ["64f1a2b3c4d5e6f7a8b9c0d1"],                     // optional, array of userIds
  "status": "draft",                                             // optional: "draft"|"completed"|"inprogress"
  "settings": {                                                  // optional
    "notifications": {
      "email": true,
      "slack": false,
      "inApp": true
    },
    "automation": {
      "autoAssignTasks": false,
      "dueDateReminders": true,
      "escalateOverdueTasks": false
    }
  }
}

----------------------------------------------------------------
3.2  UPDATE PROCESS
----------------------------------------------------------------
Method : PATCH
Route  : http://localhost:5000/api/v1/process/update/:processId
Auth   : [AUTH] [SUPERADMIN]

Headers:
  Authorization: Bearer <token>

Params:
  processId  - MongoDB ObjectId of the process (in URL)

Body (JSON) - send only fields to update (min 1):
{
  "name": "Updated Process Name",
  "status": "inprogress",
  "steps": [
    {
      "_id": "64f1a2b3c4d5e6f7a8b9c0d2",   // include _id to update existing step
      "title": "Updated Step Title",
      "status": "completed"
    },
    {
      "title": "New Step",                   // no _id = new step
      "description": "New step description"
    }
  ]
}

----------------------------------------------------------------
3.3  DELETE PROCESS

Method : DELETE
Route  : http://localhost:5000/api/v1/process/delete/:processId
Auth   : [AUTH] [SUPERADMIN]

Headers:

Body   : None

Example:

================================================================
  5. ACTIVITY LOG
================================================================
Route Prefix: /api/v1/activity-log

----------------------------------------------------------------
5.1  GET ACTIVITY LOGS (List)
----------------------------------------------------------------
Method : GET
Route  : http://localhost:5000/api/v1/activity-log/list
Auth   : [AUTH] [SUPERADMIN]

Headers:
  Authorization: Bearer <token>

Query Params (optional):
  page    - page number, default: 1
  limit   - items per page (1-100), default: 20
  action  - filter by action type (see activity actions enum)
  userId  - filter by userId (24-char hex string)

Example:
  http://localhost:5000/api/v1/activity-log/list?page=1&limit=20&userId=64f1a2b3c4d5e6f7a8b9c0d1

----------------------------------------------------------------
5.2  GET ACTIVITY LOGS BY ENTITY
----------------------------------------------------------------
Method : GET
Route  : http://localhost:5000/api/v1/activity-log/entity/:entityId
Auth   : [AUTH] [SUPERADMIN]

Headers:
  Authorization: Bearer <token>

Params:
  entityId  - MongoDB ObjectId of the entity (process, step, etc.) (in URL)

Query Params (optional):
  page   - page number, default: 1
  limit  - items per page (1-100), default: 20

Example:
  http://localhost:5000/api/v1/activity-log/entity/64f1a2b3c4d5e6f7a8b9c0d1?page=1&limit=20

Example:

================================================================
  NOTES
================================================================
- All protected routes require: Authorization: Bearer <token>
- [SUPERADMIN] routes are restricted to users with superAdmin role
- MongoDB IDs are 24-character hex strings (ObjectId format)
- Step status values  : draft | pending | completed | rejected | inreview | inprogress
- Process status values: draft | completed | inprogress
- User role values    : admin | editor | viewer
================================================================
